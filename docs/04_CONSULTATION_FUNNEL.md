# AI 상담 전환 퍼널

작성일: 2026-07-11  
목적: 무료 콘텐츠에서 AI 상담, 로그인, 광고 보상, 결제로 이어지는 사용자 흐름을 확정한다.

## 전체 흐름

```text
검색·SNS·공유 유입
  -> 오늘의 운세 / 타로 / 심리테스트
  -> 내 사주 입력
  -> 정통 사주·월간·궁합 등 무료 탐색
  -> 현재 고민 선택 또는 직접 입력
  -> 상담사 3명 추천
  -> 로그인 요청
  -> 무료 3턴 체험
  -> 광고 시청으로 1턴 추가
  -> 기본 상담 결제
  -> 프로 모드 업그레이드
  -> 상담 요약·후속 상담·재구매
```

GoatCounter에는 생년월일, 이름, 고민 원문, 상담 원문을 보내지 않는다.

## 로그인 시점

로그인은 상담 질문을 만든 뒤 요청한다. 먼저 질문을 쓰게 해야 상담 의도가 생기며, 로그인 직후에는 작성한 질문을 복원해야 한다.

서버 도입 전 정적 앱에서는:

- 상담 질문 원문을 서버로 보내지 않는다.
- 상담 CTA 클릭과 관심 이벤트만 계측한다.
- 실제 로그인/LLM/결제 UI는 모의 화면 또는 안내로 제한한다.

## 광고 보상 원칙

| 위치 | 정책 |
|---|---|
| 첫 방문 | 광고 없음 |
| 입력 화면 | 광고 없음 |
| 기본 무료 결과 | 전체 무료 |
| 추가 상세 결과 | 선택형 보상 광고 후보 |
| 로그인 화면 | 광고 없음 |
| 상담 대화 화면 | 광고 없음 |
| 무료 턴 소진 후 | 보상형 광고 선택 |
| 유료 사용자 | 광고 없음 |

광고 1회 보상은 낮게 둔다. 예: AI 1턴, 하루 최대 2회.

## 상담 화면 구조

1. AI 상담 고지
2. 상담사 선택 또는 추천 결과
3. 사용자의 질문 확인
4. 사주 근거 요약
5. 대화 영역
6. 남은 턴 또는 남은 시간
7. 안전 고지
8. 종료 요약
9. 후속 상담/프로 업그레이드 CTA
