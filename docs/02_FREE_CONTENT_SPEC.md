# 무료 콘텐츠 명세

작성일: 2026-07-11  
목적: 점신형 반복 방문 구조를 Saajuu의 정적/저위험 구현 단위로 나눈다.

## 목표

무료 콘텐츠는 유료 결과의 미리보기가 아니라 독립적으로 쓸 만해야 한다. 유료 상담은 무료 결과의 잠금 해제가 아니라, 사용자의 실제 고민을 묻고 좁히는 별도 경험이다.

## 1차 정보구조

| 메뉴 | 역할 | 초기 구현 |
|---|---|---|
| 오늘 | 매일 재방문 | 오늘의 총점, 시간대, 분야별 운세 |
| 내 사주 | 신뢰 형성 | 현재 사주 계산/성명학/공유 카드 |
| 올해의 흐름 | 시즌·월간 방문 | 2026/2027 총운, 12개월 점수 |
| 인연 | 관계 확장 | 1회성 궁합 입력과 결과 |
| 타로·테스트 | 낮은 진입 장벽 | 정적 질문형 타로/심리 테스트 |
| 상담 | 결제 전환 | 상담사 추천, 체험/유료 안내 |
| 마이 | 저장·기록 | 현재는 단일 localStorage 프로필 |

## 오늘의 운세 보고서

현재 v0.5.0.0의 `src/daily.js`를 유지하고 표현을 확장한다.

| 콘텐츠 | 설명 |
|---|---|
| 오늘의 총점 | 100점 기준 총운 |
| 한 줄 결론 | 기존 `verdict` 유지 |
| 시간대별 흐름 | 아침, 점심, 오후, 저녁, 밤 |
| 분야별 점수 | 일/관계/금전/생활 |
| 조심할 것 | 기존 `caution` 확장 |
| 행운 정보 | 숫자, 색, 방향, 음식, 아이템 |
| 오늘의 행동 | 이번 하루에 할 수 있는 작은 행동 |
| 상담 연결 질문 | 현재 운세와 고민 주제를 연결 |

## 인연·궁합

다중 프로필은 아직 보류지만, 1회성 상대 입력은 먼저 가능하다.

| 기능 | 초기 범위 |
|---|---|
| 상대 정보 입력 | 1명, 저장하지 않음 |
| 관계 선택 | 썸, 연인, 부부, 재회, 가족, 직장 |
| 기본 궁합 점수 | 무료 |
| 잘 맞는 점 | 3개 |
| 충돌하는 점 | 3개 |
| 대화 방식 | 1개 |
| 상담 CTA | 관계 고민 상담으로 연결 |

### 로그인 후 친구 연결 확장

1회성 궁합에서 사용 신호가 확인되면 계정 기반 관계 연결로 확장한다.

| 항목 | 정책 |
|---|---|
| 초대 방식 | 로그인 사용자가 불투명 초대 토큰을 생성해 공유 |
| 상대 동의 | 상대가 직접 로그인하고 자신의 출생 프로필 연결을 승인 |
| 저장 범위 | 두 사용자 ID, 관계 유형, 별칭, 동의 시각, 삭제 상태 |
| 비공개 원칙 | 생년월일시는 URL이나 공유 문구에 노출하지 않음 |
| 해제 | 어느 한쪽이 관계를 끊으면 양쪽 관계 보고서 비활성화 |
| 분석 이벤트 | 원문 생년월일·이름 없이 `relationship_invite_created`, `relationship_connected`만 기록 |

이 기능은 서버, 인증, 개인정보처리방침, 동의 로그가 준비된 뒤 착수한다.

## 타로·심리테스트

초기에는 복잡한 카드 엔진보다 정적 템플릿으로 시작한다.

| 콘텐츠 | 초기 구현 |
|---|---|
| 오늘의 한 장 | 하루 1회, 질문 선택 |
| 연애 질문 | 카드 1장 해석 |
| 선택 비교 | A/B 카드 |
| 관계 유형 테스트 | 8~10문항 |
| 직장 스트레스 유형 | 8문항 |

## 보류 항목과 관계

- PWA: 재방문 데이터 확보 전까지 보류.
- SEO 정적 페이지: 커스텀 도메인 전까지 보류.
- 다중 프로필: 정보 수정 이벤트 빈도 또는 궁합 사용 데이터 확인 후.

다만 인연/궁합은 핵심 전략이므로 다중 프로필 없이 1회성 계산으로 먼저 검증한다.
