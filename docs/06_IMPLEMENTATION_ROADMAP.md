# 구현 로드맵

작성일: 2026-07-11  
목적: GPT 챗 수익화 계획을 실제 개발 순서로 나눈다.

## 현재 기준

v0.5.0.0은 정적 PoC로 안정화되어 있다. 바로 서버/결제로 뛰기 전에, 점신 벤치마크에서 확인한 무료 콘텐츠 폭과 상담 전환 구조를 먼저 문서와 UI에 반영한다.

## Milestone 0: 기준 문서화

목표: 이후 Codex/Claude 작업자가 같은 제품 기준을 읽게 한다.

- `.gitignore`에 로컬 캡처/페르소나 원본/시크릿 보호
- `AGENTS.md` 필수 읽기 목록 갱신
- 수익화/벤치마크/무료 콘텐츠/페르소나/아키텍처 문서 생성
- `PROJECT_STATUS_README.md`에 문서 세트와 다음 작업 반영

완료 기준:

- `npm test` 통과
- `npm run build` 통과
- 문서만 변경, 런타임 동작 변경 없음

## Milestone 1: 모바일 정보구조와 무료 허브

범위:

- 오늘 / 내 사주 / 올해 / 인연 / 타로 / 상담 / 마이 구조
- 현재 사주 입력/결과 흐름 보존
- 없는 기능은 플레이스홀더 또는 정적 카드
- URL/뒤로가기/모바일 탭 UX 정리

하지 않을 것:

- 로그인, DB, 결제, LLM
- 다중 프로필
- SEO 정적 페이지

## Milestone 2: 오늘의 운세 보고서 확장

범위:

- 총점
- 시간대별 흐름
- 일/관계/금전/생활 점수
- 행운 방향/음식/아이템
- 오늘의 행동
- 상담 연결 질문

기술:

- `src/daily.js`의 기존 일진/십신/지지 관계 알고리즘 재사용
- 테스트 추가

## Milestone 3: 연간·월간·7일 운세

범위:

- 내일 운세 확장
- 7일 한 줄 운세
- 12개월 점수
- 현재 달/다음 달 상세
- 신년운세 랜딩의 내부 UI 초안

## Milestone 4: 1회성 인연·궁합

범위:

- 상대 1명 입력
- 저장하지 않는 궁합 계산
- 잘 맞는 점/충돌점/대화법
- 미선/준호/성우 상담 추천 CTA
- 로그인 후 공유 초대로 서로 친구처럼 묶는 관계 연결은 서버/계정 단계에서 설계만 보존

게이트:

- 다중 프로필은 여전히 보류. 정보 수정 이벤트나 궁합 사용 데이터가 쌓인 뒤 착수.
- 친구 연결은 양쪽 사용자 동의, 초대 토큰, 관계 해제, 출생정보 비노출 정책이 준비된 뒤 착수.

## Milestone 5: 타로·심리 테스트

범위:
